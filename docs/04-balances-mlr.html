What an address retains — and whether fraud changes the pattern

An address’s balance is a behavioural quantity, not just an accounting one. Two addresses can move identical volumes of Ether and end up in very different places: one accumulates, the other passes everything through. In fraud analytics that distinction matters — pass-through behaviour is the signature of mule-like accounts — so this chapter asks a precise version of the question: conditional on how much an address sends and how often it transacts, does anything remain? And is that pattern different for fraudulent addresses — not just in level, but in shape?

Two linear models answer it: a main-effects model, and one that lets the sent→balance slope differ by fraud status.

Setting up the analysis

Code 
library(dplyr)
library(ggplot2)
library(broom)
library(car)

trxs <- readRDS("data/eth_clean.rds")

mlr <- trxs %>%
  filter(is.finite(total_ether_balance), total_ether_balance >= 0) %>%
  mutate(
    balance_log = log10(total_ether_balance + 1),
    sent_log    = log10(total_ether_sent + 1),
    tnx_log     = log10(total_transactions + 1)
  )

Two preparation decisions shape everything downstream. First, 978 addresses carry a negative recorded balance — an artefact of how the aggregate was computed at extraction time rather than a possible state of an account — and are excluded, at the cost of about 10% of the sample (n = 8863 remains). Second, every continuous variable in this dataset is heavily right-tailed, so the analysis works on log10(x + 1) scales. That choice buys well-behaved models but changes how coefficients read: each coefficient describes what a ×10 change in a predictor does, multiplicatively, to (balance + 1) — a reading guide worth keeping in mind for every table below.

Three effects, all large and all interpretable. Holding the others fixed:

  • Transaction count builds balance. A ×10 increase in total transactions is associated with multiplying (balance + 1) by 3.2 — roughly +222%. Busier addresses hold more, which is what accumulation looks like.
  • Outflow drains it — even conditionally. At a fixed transaction count, a ×10 increase in total Ether sent is associated with a -48% change in (balance + 1). The sign is the substantive finding: once you control for how often an address transacts, sending more is pass-through, not throughput.
  • Fraudulent addresses retain less. At the same activity level, a flagged address is associated with -42% in (balance + 1) relative to a legitimate one — consistent with accounts built to move value on rather than to hold it.

Together the three predictors account for 35% of the variance in log-balance — a solid share for three aggregates, while leaving plenty of address-level behaviour unexplained.

Code 
ncv_p <- ncvTest(m1)$p
cooks_flagged <- sum(cooks.distance(m1) > 4 / nrow(mlr))

The diagnostics say: directionally trustworthy, precisely optimistic. A formal check confirms what the residual plot shows — non-constant variance (Breusch–Pagan p < 0.001) — which means classical standard errors are somewhat too confident. That caveat is harmless for the three main effects (|t| ≥ 14; their direction and significance are not in doubt) but it matters for borderline estimates, a point that becomes relevant in the next section. Influence checks flag 491 observations (5.5% of the sample) by the 4/n Cook’s-distance rule — genuine extreme-but-real addresses, retained rather than trimmed.

Does fraud bend the relationship, or just shift it?

Model 1 says fraudulent addresses hold less on average. The sharper question is whether the slope differs — whether each additional unit of outflow costs a fraudulent address more (or less) of its balance than a legitimate one. Model 2 adds that interaction.

m2 <- lm(balance_log ~ sent_log * flag_f + tnx_log, data = mlr)

       colour = NULL) +

The interaction estimate is -0.044 with p = 0.044 — under the conventional threshold, but only just, and this is exactly where two earlier observations converge: with n = 8,863, effects this small reach significance easily, and the heteroscedasticity noted above makes borderline p-values the least reliable numbers in the chapter.

Choosing between the models

Code 
tibble(model = c("M1: main effects", "M2: + sent × fraud"),
       adj_R2 = c(summary(m1)$adj.r.squared, summary(m2)$adj.r.squared),
       AIC = c(AIC(m1), AIC(m2)),
       `F-change p` = c(NA, cmp$`Pr(>F)`[2])) %>%
  knitr::kable(digits = 4, caption = "Model comparison — statistically detectable, practically negligible.")
Model comparison — statistically detectable, practically negligible.
model adj_R2 AIC F-change p
M1: main effects 0.3535 16490.28 NA
M2: + sent × fraud 0.3538 16488.22 0.044

Statistical significance and practical importance part ways here, and the comparison table is the referee: the interaction improves adjusted R² by 0.0002 and AIC by about 2 points. A slope difference that small changes no substantive conclusion, so Model 1 is the model to keep: fraud status shifts the level of retained balance clearly, but there is no meaningful evidence that it changes the shape of the outflow–balance relationship.

What this chapter established

Balance behaves like a behavioural fingerprint: activity accumulates it, outflow drains it conditionally, and fraudulent addresses systematically retain less — about -42% in (balance + 1) at equal activity — with no practically meaningful interaction on top. The estimates describe cross-sectional aggregates on multiplicative (log) scales, exclude the ~10% of addresses with artefactual negative balances, and inherit the usual caution that heavy-tailed data place on precise intervals; none of those caveats touches the direction or substance of the three main effects.

## Choosing between the models

```{r compare}
tibble(model = c("M1: main effects", "M2: + sent × fraud"),
       adj_R2 = c(summary(m1)$adj.r.squared, summary(m2)$adj.r.squared),
       AIC = c(AIC(m1), AIC(m2)),
       `F-change p` = c(NA, cmp$`Pr(>F)`[2])) %>%
  knitr::kable(digits = 4, caption = "Model comparison — statistically detectable, practically negligible.")
```

Statistical significance and practical importance part ways here, and the
comparison table is the referee: the interaction improves adjusted R² by
`r sprintf("%.4f", summary(m2)$adj.r.squared - summary(m1)$adj.r.squared)` and AIC
by about `r sprintf("%.0f", AIC(m1) - AIC(m2))` points. A slope difference that
small changes no substantive conclusion, so **Model 1 is the model to keep**: fraud
status shifts the level of retained balance clearly, but there is no meaningful
evidence that it changes the *shape* of the outflow–balance relationship.
